    Анализ проблемы

    • Если изображение зашумлено, возможно, потребуется фильтрация
      • Bilateral
      • Median
    • Увеличь контраст изображения.
      • CLAHE
    • Если это скрин интерфейса, сайта или игры, то, вероятно, текст и другие элементы выровнены и можно применить простые и быстрые алгоритмы.
    • Если скрин экрана без антиалиасинга (в некоторых играх можно отключить), то можно применить совсем элементарные и быстрые алгоритмы наподобие PixelOCR, hit-or-miss.
    • Если объект имеет фиксированный размер и наклон, то используй templatematch.
    • Цветное ли изображение? Можно ли окрасить объект для более простого опознания?
      • inRange
      • backprojection
      • thresholding
      • RGB Euclidian distance + thresholding
    • Есть ли мелкие детали? Их можно извлечь с помощью top-hat, black-hat, DOG
    • Есть ли устойчивые контурные очертания для контурного анализа?
    • Есть ли устойчивые паттерны из нескольких блобов?
    • Есть ли постоянные соседствующие цвета? Их можно продетектировать с помощью inRange, применить dilate и операцию AND. По получившимся полосам найти регионы интереса.
    • Для снижения числа и размеров блобов можно применить dilate и/или erosion для сокрытия линий и других элементов. Будет проще найти регионы интереса.
    • В случае скриншотов почти всегда бывают всплески на гистограммах. Это можно использовать для сегментации и предварительной оценки изображений.
    • Есть ли контуры с плавно спадающими магнитудами?
    • С помощью прямого и обратного преобразования Фурье можно удалить определённые частоты на изображении для упрощения последующего анализа.
    • Особую ценность в контурном анлизе представляют блобы с дырками внутри. Один кластер с тремя контурами может легко ограничить диапазон поиска образа (8, B, etc).
